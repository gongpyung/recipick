# 01. PRD

## 제품 한 줄
유튜브 일반 영상/쇼츠 URL을 입력하면 AI가 레시피를 구조화해서 보여주고, 사용자가 인분 수를 바꿨을 때 재료량이 자동 조정되는 웹앱.

---

## 해결하려는 문제
요리 영상을 볼 때 반복되는 불편이 있다.

- 영상 링크를 나중에 다시 찾기 어렵다.
- 쇼츠/짧은 영상은 재료와 순서를 놓치기 쉽다.
- 영상을 계속 돌려보며 요리해야 한다.
- 1인분/2인분/4인분 등 인분을 바꾸면 재료량을 다시 계산해야 한다.
- 영상 내용이 압축되어 있어, 실제 조리 시 텍스트 레시피가 더 필요하다.

---

## 핵심 사용자
### 1차 타깃
- 집에서 요리할 때 유튜브 레시피를 자주 참고하는 사람
- 영상보다 텍스트 레시피를 선호하는 사람
- 2인분 레시피를 1인/3인/4인으로 조절하는 일이 잦은 사람
- 가족/배우자가 같은 레시피를 다시 보고 재현하고 싶은 사람

### 대표 시나리오
1. 사용자가 유튜브 링크를 붙여넣는다.
2. 시스템이 레시피를 구조화한다.
3. 사용자는 재료/단계/팁을 텍스트로 확인한다.
4. 인분 수를 바꾸면 재료량이 자동 조정된다.
5. AI 결과가 애매하면 사용자가 직접 수정한다.

---

## MVP 목표
### 반드시 포함
1. 유튜브 링크 → 구조화된 레시피 추출
2. 인분 수 변경 → 숫자형 재료량 자동 계산
3. 결과 수정
4. 최근 생성 결과 다시 보기

### 제외 또는 후순위
- 공개 레시피 피드
- 장보기 리스트
- 영양정보
- 공유/협업 기능
- 다국어 번역
- 유튜브 외 플랫폼 링크 지원

---

## 핵심 가치 제안
- **영상 다시 보기 없이 바로 요리 가능**
- **인분 조절이 쉬움**
- **AI 결과를 믿고 끝내는 게 아니라 직접 수정 가능**
- **실제 조리에 적합한 텍스트 중심 경험**

---

## 기능 범위
### 입력
- 유튜브 일반 영상 URL
- 유튜브 쇼츠 URL
- URL 유효성 검사

### 추출
- 제목
- 기본 인분 수
- 재료 목록
- 조리 단계
- 팁/메모
- 경고/불확실성 표시

### 조절
- 1~8인분 조정
- 숫자형 재료량 자동 배율 조정
- 비정형 표현 유지 (`약간`, `적당량`, `한 줌` 등)

### 편집
- 재료명/수량/단위 수정
- 단계 수정
- 기본 인분 수정

### 보존
- 최근 생성 결과 다시 보기
- 새로고침 수준의 기본 보존

---

## 성공 기준
### 정량
- 구조화 결과 schema valid 비율 95% 이상
- 추출 성공률 80% 이상 (지원 대상 영상 기준)
- 인분 조절 계산 오류 치명 이슈 0건
- 평균 첫 결과 표시 시간 20초 이내

### 정성
- “영상을 다시 안 봐도 되겠다”는 느낌이 드는가
- 사용자가 재료/단계를 한 화면에서 이해할 수 있는가
- 가족이 대신 봐도 재현 가능한 수준인가

---

## 주요 리스크
1. 자막 없는 영상 비율
2. 분량이 모호한 영상 (`적당량`, `조금`, `취향껏`)
3. 쇼츠 특성상 정보 누락이 많음
4. 레시피가 아닌 먹방/브이로그/광고 영상 구분 필요
5. 재료명/단위 정규화 난이도

---

## 제품 방향에 대한 권고
**첫 MVP는 자막/설명 중심으로 가고, OCR/비전은 후속으로 미루는 것을 권장한다.**
이유:
- 첫 버전 품질 통제가 쉬움
- 구현 복잡도와 비용이 낮음
- 평가셋 구축이 단순해짐
- 인분 조절/편집 UX를 먼저 완성할 수 있음

---

## 구현 가정 (2026-03-20 반영)
- 1차 LLM API 제공자는 **Z.ai**
- 사용할 상품/접속 기준은 **GLM 코딩 플랜**
- 앱은 브라우저에서 직접 LLM API를 호출하지 않고, 서버 API를 통해 Z.ai를 호출한다.
- 모델명은 코드에 박지 않고 환경변수(`ZAI_MODEL`)로 관리한다.
