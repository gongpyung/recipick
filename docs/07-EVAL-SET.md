# 07. Eval Set

## 목적

이 프로젝트는 AI 결과 품질이 핵심이므로, “대충 되는 것 같음” 상태를 피하기 위해 초기에 평가셋을 만든다.

---

## 평가 관점

1. schema valid 여부
2. 핵심 재료 누락 여부
3. 환각 재료 추가 여부
4. 기본 인분 추출 여부
5. 단계 분리 품질
6. 불확실성 표시 정직성
7. unsupported 영상 탐지 정확도

---

## 목표 지표

- schema valid rate >= 95%
- 핵심 재료 recall >= 80%
- 환각 재료 비율 <= 10%
- unsupported 영상 오탐률 <= 10%

---

## 테스트 케이스 카테고리

### A. 자막이 풍부한 일반 요리 영상

- 기대: 높은 정확도

### B. 쇼츠

- 기대: 재료/단계 일부 누락 가능, warning 필요

### C. 설명란은 풍부하지만 자막은 빈약한 영상

- 기대: partial success

### D. 자막이 없는 영상

- MVP에서는 fail 또는 unsupported 허용

### E. 분량이 거의 없는 영상

- 기대: 경고 다수 생성

### F. 두 가지 요리가 섞인 영상

- 기대: `MULTIPLE_DISHES_DETECTED`

### G. 먹방/브이로그/광고

- 기대: `NON_RECIPE_VIDEO`

### H. 영어 단위 영상 (`cup`, `tbsp`)

- 기대: 단위 정규화

### I. 한국식 모호 표현 (`적당량`, `한 줌`, `취향껏`)

- 기대: `scalable=false`

### J. 재료명은 있는데 수량이 없는 영상

- 기대: `MISSING_QUANTITY`

---

## 케이스 템플릿

```md
### Case 01

- URL:
- 타입: video | shorts
- 설명:
- 기대 결과:
  - 필수 재료:
  - 기본 인분:
  - unsupported 여부:
  - 주요 warning:
```

---

## 수동 점검 체크리스트

- 제목이 맞는가?
- 기본 인분이 과하게 추정되지 않았는가?
- 핵심 재료가 빠지지 않았는가?
- 없는 재료를 지어내지 않았는가?
- 단계가 실제 조리 순서로 읽히는가?
- 애매한 부분을 warning으로 정직하게 드러냈는가?

---

## 권장 운영 방식

- 처음엔 15~20개 URL만 선정
- 기능 추가 전후 동일 eval set으로 비교
- raw output / normalized output / 최종 UI 표시 결과를 함께 검토
